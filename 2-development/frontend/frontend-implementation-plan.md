# План создания веб-интерфейса для MVP

## Цель

На основе аналитики дизайна и backend-контрактов разработать веб-интерфейс для клиента, который позволит:

- [x] реализовать экраны регистрации и входа;
- [x] показать расписание тренировок на 7 дней вперёд в интерфейсе;
- [x] реализовать экран бронирования;
- [x] реализовать экран списка и деталей броней;
- [x] открыть правила посещения и настройки уведомлений.

## Что уже сделано

- [x] созданы основные страницы по аналитике: регистрация, вход, расписание, бронирование, мои бронирования, детали, инфо-центр, уведомления;
- [x] добавлена навигация между ключевыми маршрутами;
- [x] интерфейс оформлен в стиле MVP без лишних элементов и по структуре аналитических экранов;
- [x] вход и базовая защита приватных маршрутов подключены к браузерной сессии;
- [x] расписание, бронирование, мои бронирования и настройки уведомлений интегрированы с backend API;
- [x] исправлена логика отображения расписания: даты формируются от текущего дня, а не по жёстко заданному набору значений.

## Базовый стек

- [x] Next.js + TypeScript
- [x] Tailwind CSS
- [x] React Query / TanStack Query
- [x] App Router
- [x] REST API через backend

## Принцип работы

План строится по принципу итеративной разработки:

1. [x] сначала делаем базовый каркас и основной пользовательский путь;
2. [x] затем добавляем детали, состояния ошибок и UX-поддержку;
3. [x] после каждой итерации проверяем сценарий на реальном API и вносим правки.

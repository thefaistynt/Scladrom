# Онбординг проекта: клиентское мобильное приложение для скалодрома

## 1. Контекст проекта

Проект направлен на создание мобильного приложения для клиентов скалодрома «Вертикаль». Основная задача — дать пользователю возможность самостоятельно находить доступные тренировки, записываться на них, управлять своими бронями и видеть правила посещения.

## 3. Что входит в текущую разработку

В рамках текущей поставки разрабатываем:
- клиентское мобильное приложение;
- API-контракт и клиентскую интеграцию с существующим бэкендом;
- сценарии только для роли Client.

## 4. Что не входит в MVP

Не нужно реализовывать в текущем спринте:
- интерфейсы инструктора и администратора;
- создание/редактирование расписания из приложения;
- онлайн-оплату;
- рейтинг инструктора (post-MVP);
- расширенную аналитику и админские сценарии.

## 5. Ключевые функции MVP

### Авторизация и регистрация
- регистрация по email и паролю;
- обязательный возраст 18+;
- принятие правил безопасности/оферты при первой записи.

### Просмотр расписания
- список слотов на 7 дней вперёд;
- фильтры по формату, инструктору, времени суток и доступности;
- empty state при отсутствии слотов.

### Бронирование
- выбор слота;
- выбор количества мест;
- выбор варианта снаряжения: прокат или своё;
- отображение цены тренировки и проката;
- правила отмены до подтверждения;
- атомарная проверка доступности мест на стороне сервера.

### Управление бронями
- список активных и отменённых броней;
- бесплатная отмена не позднее чем за 24 часа;
- блокировка отмены для поздних случаев;
- обработка статуса «Отменена скалодромом».

### Уведомления и информация
- push-напоминания;
- push-уведомления об отмене бронирования;
- просмотр правил посещения и информации по скалодрому.

## 6. Бизнес-правила, которые критичны для реализации

- Нельзя допускать двойные брони: защита обеспечивается бэкендом, приложение должно корректно обрабатывать ошибку 409 Conflict.
- При отмене слота скалодромом бронь не удаляется, а получает статус «Отменена скалодромом».
- Повторная запись на отменённый слот запрещена.
- Для новичка запрещены слоты формата «трассы с верёвкой».
- Частичный прокат не поддерживается.

## 7. Данные и контракт API

Приложение работает как клиент к внешнему API. Ключевые сущности:
- Client
- TrainingSlot
- Booking
- EquipmentRental
- Instructor
- TrainingFormat
- Zone
- InstructorRating

Основной контракт находится в [1-analytics/api/openapi.yaml](../1-analytics/api/openapi.yaml).

## 8. Рекомендуемый порядок разработки

1. Подготовка проекта и базовой архитектуры приложения.
2. Реализация авторизации и регистрации.
3. Экран списка слотов и фильтров.
4. Экран деталей слота и создания брони.
5. Экран «Мои брони» и сценарии отмены.
6. Интеграция уведомлений и правил посещения.
7. Тестирование и дотягивание UX до уровня MVP.

## 9. Критерии готовности MVP

- пользователь может зарегистрироваться;
- пользователь видит доступные слоты на 7 дней;
- пользователь может забронировать слот;
- пользователь может отменить бронь в допустимое время;
- приложение корректно обрабатывает ошибки конфликтов и отмены слота;
- приложение соответствует основным требованиям из функциональной спецификации.

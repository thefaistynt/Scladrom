# План реализации backend части приложения

## Принцип

План строится по принципу итеративной разработки и соответствует аналитике: делаем только клиентский сценарий, работаем через внешний/существующий бэкенд как источник истины, реализуем бизнес-правила и контракт API для мобильного приложения.

> Состояние на 2026-07-06: проверено по фактическому коду backend и ключевым регрессионным тестам авторизации. Тесты: `python -m pytest tests/test_booking_flow.py -k "login"` → 2 passed.

## Критерии готовности backend MVP

- [x] клиент может зарегистрироваться и войти
- [x] клиент видит доступные слоты на 7 дней
- [x] клиент может забронировать слот
- [x] клиент может отменить бронь в допустимое время
- [x] backend корректно обрабатывает конфликт мест и отмену слота скалодромом
- [x] API соответствует требованиям и ограничениям из аналитики

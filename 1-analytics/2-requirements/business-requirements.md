# Бизнес-требования — скалодром «Вертикаль» (клиентское мобильное приложение)

> **Источник:** [domain.md](../1-elicitation/domain.md)
> **Приоритеты:** MoSCoW — **Must** (обязательно для MVP), **Should** (желательно), **Could** (post-MVP), **Won't** (вне скоупа)

| ID | Формулировка | Приоритет | Источник |
|----|--------------|-----------|----------|
| BZ-001 | Клиенты должны иметь возможность самостоятельно записываться на групповые тренировки через мобильное приложение, без участия персонала | Must | §1 «Цель» |
| BZ-002 | Владелец должен контролировать процесс записи через уже существующую инфраструктуру (админку и интерфейс инструктора), без создания новых интерфейсов для них | Must | §1 «Цель»; §9 |
| BZ-003 | Система должна исключить двойные брони и путаницу с занятостью групп, возникающие при ручном учёте через Telegram и бумажную тетрадь | Must | §1 «Проблема»; §8 |
| BZ-004 | Приложение должно быть готово к запуску нового сезона (начало осени) | Must | §1 «Дедлайн» |
| BZ-005 | Приложение должно быть доступно на платформах iOS и Android | Must | §9; R-028 |

# Функциональные требования — скалодром «Вертикаль» (клиентское мобильное приложение)

> **Источник:** [domain.md](../1-elicitation/domain.md)
> **Приоритеты:** MoSCoW — **Must** (обязательно для MVP), **Should** (желательно), **Could** (post-MVP), **Won't** (вне скоупа)

| ID | Формулировка | Приоритет | Источник |
|----|--------------|-----------|----------|
| FR-001 | Система должна позволять клиенту зарегистрироваться по email и паролю | Must | §4.1; §5.1; BR-011 |
| FR-002 | Система должна допускать регистрацию только совершеннолетних клиентов (18+) | Must | §4.1; BR-011 |
| FR-003 | Система должна требовать принятия правил безопасности/оферты при первой записи клиента | Must | §4.1; §5.1; §7 |
| FR-004 | Система должна предоставлять клиенту документ с правилами посещения (адрес, режим, что взять) | Must | §5.1 |
| FR-005 | Система должна отображать клиенту список слотов тренировок на 7 дней вперёд по умолчанию | Must | §4.2; §5.2; BR-005 |
| FR-006 | Система должна позволять расширить диапазон отображаемых слотов через фильтр дат | Should | §4.2 |
| FR-007 | Система должна предоставлять фильтры по формату тренировки, инструктору, времени суток и наличию свободных мест | Must | §4.2; §7 |
| FR-008 | Система должна показывать сообщение-заглушку («Пока нет доступных тренировок»), если слоты отсутствуют | Must | §5.2 |
| FR-009 | Система должна позволять клиенту выбрать готовый слот (время, формат и инструктор фиксированы вместе, отдельно инструктор не выбирается) | Must | §4.2; §5.3 |
| FR-010 | Система должна позволять клиенту указать количество бронируемых мест (запись с друзьями) | Must | §4.3; §5.3; BR-012 |
| FR-011 | Система должна разрешать клиенту иметь несколько одновременных активных бронирований | Must | §4.3 |
| FR-012 | Система должна разрешать клиенту с опытным уровнем записываться на слоты в новичковой зоне | Must | §4.3; BR-013 |
| FR-013 | Система должна запрещать клиенту-новичку записываться на слоты формата «трассы с верёвкой» (опытный уровень) | Must | §4.3; BR-013 |
| FR-014 | Система должна позволять клиенту выбрать вариант снаряжения: полный комплект проката либо полностью своё снаряжение | Must | §4.4; §5.3; BR-014 |
| FR-015 | Система не должна поддерживать частичный прокат снаряжения | Must | §4.4; BR-014 |
| FR-016 | Система должна показывать стоимость проката отдельной строкой от стоимости тренировки | Must | §4.4; §5.3 |
| FR-017 | Система должна показывать итоговую сумму (тренировка + прокат) до подтверждения записи | Must | §4.4; §5.3; BR-016 |
| FR-018 | Система должна выводить предупреждение, если прокатного фонда недостаточно, и позволять клиенту подтвердить запись на свою ответственность | Must | §4.4; BR-015 |
| FR-019 | Система должна показывать клиенту правила отмены перед подтверждением записи | Must | §5.3; BR-016 |
| FR-020 | Система должна атомарно проверять наличие свободных мест на бэкенде при подтверждении записи | Must | §5.3; BR-006 |
| FR-021 | Система должна выводить сообщение «место уже занято» при конфликте бронирования, без постановки в очередь и автоподбора другого слота | Must | §5.3 |
| FR-022 | Система должна позволять клиенту бесплатно отменить бронирование не позднее чем за 24 часа до начала тренировки | Must | §4.3; §5.4; BR-009 |
| FR-023 | Система должна блокировать возможность отмены (кнопка недоступна), если до начала тренировки осталось менее 24 часов | Must | §5.4; BR-009 |
| FR-024 | Система не должна выделять неявку (no-show) в отдельный статус — применяется то же правило, что и для поздней отмены | Must | §4.3; §5.4; BR-018 |
| FR-025 | Система должна отражать статус бронирования «Отменена скалодромом» с указанием причины при отмене администратором | Must | §4.3; §5.5; BR-007 |
| FR-026 | Система должна отправлять клиенту push-уведомление с извинениями при отмене тренировки скалодромом | Must | §5.5; BR-007 |
| FR-027 | Система должна запрещать повторную запись клиента на слот, отменённый скалодромом | Must | §5.5; BR-008 |
| FR-028 | Система должна отправлять клиенту push-напоминание о тренировке с настраиваемым временем отправки | Must | §5.6 |
| FR-029 | Система должна отправлять клиенту push-уведомление при отмене бронирования (клиентом или скалодромом) | Must | §5.6 |
| FR-030 | Система не должна отправлять push-уведомления о подтверждении записи, малом количестве свободных мест или смене инструктора | Must | §5.6 |
| FR-031 | Система должна позволять клиенту оставить оценку инструктору (1–5) с текстовым отзывом в течение 24 часов после тренировки | Could (post-MVP) | §4.5; §5.7; §7 |
| FR-032 | Система должна запрещать изменение оценки инструктора после её отправки | Could (post-MVP) | §4.5 |
| FR-033 | Система не должна создавать и не должна редактировать слоты расписания из клиентского приложения | Must (ограничение скоупа) | §9; BR-010 |
| FR-034 | Система должна информировать клиента о стоимости тренировки и проката; фактическая оплата производится на месте (без онлайн-оплаты в MVP) | Must | §5.3; BR-019 |
| FR-035 | Система должна поддерживать онлайн-оплату при записи с возвратом средств при отмене в срок | Could (post-MVP) | BR-020; §7 |
| FR-036 | Система не должна показывать клиентам средний рейтинг инструктора | Could (post-MVP, вместе с FR-031) | §4.5 |

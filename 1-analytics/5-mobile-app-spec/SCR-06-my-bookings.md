# SCR-06 Мои бронирования

**ID:** SCR-06  
**Тип:** Экран  
**Домен:** Брони  
**Приоритет:** Critical  
**Статус:** Черновик  
**Зона авторизации:** АЗ  

---

## Обзор

Экран списка всех броней клиента. Здесь пользователь видит активные записи, историю и текущий статус каждой брони.

### User Story

> Как клиент, я хочу видеть свои бронирования, чтобы легко контролировать предстоящие тренировки и их статус.

### Бизнес-ценность

- даёт владельцу контроля над собственными записями;
- упрощает отмену и поиск нужной тренировки;
- снижает число звонков и переписок по статусу броней.

## Входные данные

| Название | Тип | Описание |
|---|---|---|
| bookings | array | Список броней клиента |
| selectedTab | string | Active / History |

---

## Элементы экрана

- активные брони;
- архивные/отменённые брони;
- бейджи статусов;
- empty state при отсутствии записей;
- переход к деталям брони.

---

## Состояния экрана

| Состояние | Поведение |
|---|---|
| Loading | Показать loader |
| Loaded | Показать список |
| Empty | Показать empty state |
| Error | Показать ошибку и кнопку «Повторить» |

---

## Действия пользователя

1. Открыть список броней.
2. Переключиться между активными и архивными.
3. Открыть детали брони.

---

## Используемые запросы

### GET /bookings

**Триггер:** Открытие экрана и обновление.

**Обработка ответа:**

- success → показать список броней;
- empty → показать empty state;
- error → показать сообщение.

## Критерии приёмки

- Дано: клиент имеет активную бронь, **Когда** он открыл экран, **Тогда** она отображается в разделе активных.
- Дано: бронь была отменена скалодромом, **Когда** клиент открыл список, **Тогда** отображается статус «Отменена скалодромом» и причина.

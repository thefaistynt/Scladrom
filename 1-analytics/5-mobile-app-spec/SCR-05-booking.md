# SCR-05 Бронирование тренировки

**ID:** SCR-05  
**Тип:** Экран  
**Домен:** Бронирование  
**Приоритет:** Critical  
**Статус:** Черновик  
**Зона авторизации:** АЗ  

---

## Обзор

Экран оформления записи на выбранный слот. На этом экране клиент выбирает количество мест, вариант снаряжения и подтверждает бронь.

### User Story

> Как клиент, я хочу оформить запись на тренировку, чтобы занять место в группе и подготовиться к визиту.

### Бизнес-ценность

- обеспечивает основной сценарий записи;
- даёт клиенту прозрачность по стоимости и правилам;
- уменьшает количество ошибок при ручной записи.

## Входные данные

| Название | Тип | Описание |
|---|---|---|
| slotId | string | ID выбранного слота |
| slotDetails | object | Дата, время, инструктор, формат |
| rentalOption | string | own / rental |
| seatsCount | integer | Количество мест |
| acceptSafetyRules | boolean | Согласие с правилами |
| rentalAvailability | object | Доступность проката |

---

## Элементы экрана

- блок с информацией о тренировке;
- выбор количества мест;
- переключатель «Своё снаряжение / Полный прокат»;
- блок стоимости тренировки и проката;
- итоговая сумма;
- правила отмены;
- предупреждение при нехватке прокатного фонда;
- кнопка «Подтвердить запись».

---

## Состояния экрана

| Состояние | Поведение |
|---|---|
| Draft | Показать форму с текущими значениями |
| Rental warning | Показать предупреждение о нехватке проката |
| Confirming | Заблокировать UI, показать loader |
| Success | Перейти в список броней |
| Conflict | Показать ошибку «Место уже занято» |

---

## Действия пользователя

1. Проверить детали слота.
2. Выбрать количество мест.
3. Выбрать вариант снаряжения.
4. При необходимости подтвердить предупреждение по прокату.
5. Нажать «Подтвердить запись».

---

## Используемые запросы

### GET /slots/{slotId}

**Триггер:** Открытие экрана.

**Обработка ответа:** показать детали слота и вычислить итоговую стоимость.

### POST /bookings

**Триггер:** Нажатие кнопки подтверждения.

**Параметры:** slotId, seatsCount, rentalOption.

**Обработка ответа:**

- success → создать бронь и перейти к списку броней;
- 409 → показать «Место уже занято»;
- 410 → показать «Слот отменён или больше недоступен»;
- сеть → показать сообщение об ошибке.

## Критерии приёмки

- Дано: клиент выбрал слот и указал количество мест, **Когда** он подтверждает запись, **Тогда** система создаёт бронь только если есть свободные места.
- Дано: прокатный фонд недостаточен, **Когда** клиент нажимает подтвердить, **Тогда** система показывает предупреждение и даёт возможность продолжить на свою ответственность.
- Дано: место уже занято, **Когда** запрос отправлен, **Тогда** система показывает сообщение о конфликте и не создаёт бронь.

# SCR-01 Регистрация клиента

**ID:** SCR-01  
**Тип:** Экран  
**Домен:** Авторизация  
**Приоритет:** Critical  
**Статус:** Черновик  
**Зона авторизации:** НЗ  

---

## Обзор

Экран регистрации нового клиента. Используется для создания учётной записи и получения доступа к функционалу записи на тренировки.

### User Story

> Как клиент, я хочу зарегистрироваться по email и паролю, чтобы получить доступ к записи на тренировки.

### Бизнес-ценность

- упрощает первый вход в приложение;
- исключает ручную регистрацию со стороны персонала;
- даёт доступ к бронированию без участия администратора.

## Входные данные

| Название | Тип | Описание |
|---|---|---|
| email | string | Email клиента |
| password | string | Пароль |
| birthDate | date | Дата рождения |
| acceptSafetyRules | boolean | Подтверждение 18+ и принятие правил |

---

## Элементы экрана

- поле email;
- поле password;
- чекбокс подтверждения совершеннолетия;
- кнопка «Зарегистрироваться»;
- ссылка на вход.

---

## Состояния экрана

| Состояние | Условие | Поведение |
|---|---|---|
| Idle | Экран открыт | Показать форму |
| Validation error | Невалидные данные | Показать inline-ошибки |
| Loading | Отправка запроса | Заблокировать форму, показать loader |
| Success | Регистрация прошла | Сохранить токен и перейти в расписание |
| Error | 4xx/5xx/сеть | Показать сообщение об ошибке |

---

## Действия пользователя

1. Ввести email и пароль.
2. Указать дату рождения.
3. Подтвердить возраст и правила безопасности.
4. Нажать «Зарегистрироваться».
5. При необходимости перейти на экран входа.

---

## Используемые запросы

### POST /auth/register

**Триггер:** Нажатие кнопки регистрации.

**Параметры:** email, password, birthDate, acceptSafetyRules.

**Обработка ответа:**

- success → сохранить accessToken и перейти в расписание;
- 400 → показать причины валидации;
- ошибка сети → показать сообщение «Проверьте подключение».

## Критерии приёмки

- Дано: пользователь заполнил все поля, **Когда** он нажимает «Зарегистрироваться», **Тогда** система отправляет запрос и создаёт аккаунт.
- Дано: возраст не подтверждён, **Когда** пользователь пытается зарегистрироваться, **Тогда** регистрация запрещена.
- Дано: email имеет неверный формат, **Когда** пользователь нажимает кнопку, **Тогда** форма показывает ошибку.

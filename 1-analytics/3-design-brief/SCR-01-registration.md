# Постановка на дизайн: SCR-01 Экран регистрации

## Описание
Экран для создания новой учетной записи клиента. Должен быть максимально простым и быстрым.

## Цель
Позволить пользователю зарегистрироваться в системе для доступа к бронированию тренировок.

## Функциональные элементы
- **Поле ввода Email**: Валидация формата email.
- **Поле ввода Пароля**: Скрытие символов, проверка сложности (согласно стандартам безопасности).
- **Чекбокс подтверждения возраста**: «Мне исполнилось 18 лет» (обязательно для регистрации).
- **Кнопка «Зарегистрироваться»**: Активна только при заполнении всех полей и установке чекбокса.
- **Ссылка «Уже есть аккаунт? Войти»**: Переход на экран авторизации (SCR-02).

## Требования к UX/UI
- Минималистичный интерфейс.
- Четкие сообщения об ошибках валидации (например, «Некорректный формат email»).
- Акцент на кнопке основного действия.

## Связанные требования
- FR-001, FR-002, UC-001, US-001
